**No-Zero integer** is a positive integer that **does not contain any `0`** in its decimal representation.

Given an integer `n`, return _a list of two integers_ `[a, b]` _where_:

- `a` and `b` are **No-Zero integers**.
- `a + b = n`

The test cases are generated so that there is at least one valid solution. If there are many valid solutions, you can return any of them.

**Example 1:**

**Input:** n = 2
**Output:** [1,1]
**Explanation:** Let a = 1 and b = 1.
Both a and b are no-zero integers, and a + b = 2 = n.

**Example 2:**

**Input:** n = 11
**Output:** [2,9]
**Explanation:** Let a = 2 and b = 9.
Both a and b are no-zero integers, and a + b = 11 = n.
Note that there are other valid answers as [8, 3] that can be accepted.

**Constraints:**

- `2 <= n <= 104`

---
## Solution

You can do this in $O(log_{10}(n))$ time complexity and $O(1)$ space by dividing and splitting each digit per digit moving from right to left. 

In cases where the digit you consider is 2 or greater, you can simply decompose into $digit - 1$ and $1$.

In cases where the digit equals $1$, either 
- the overall number to decompose is less than $10$, in which case we just have the remainder $1$ which can be added to either number we decompose into, or 
- the number is greater than $10$ and we need to borrow from one position to the left (e.g. $91, 81, 671,...$ ) and such cases can be broken down into $9+2$ after borrowing a $1$ from a position to the left

Similarly, in cases where the digit equals $0$, it's either done or we need to decompose into a $9 + 1$ after borrowing from the left place if the number is greater than $10$.

```py
def getNoZeroIntegers(self, n: int) -> List[int]:
    n1, n2 = 0, 0
    pos = 1

    while n > 0:
        rem = n % 10
        if rem == 0: # n must be greater than 10 so we can borrow from the left
            n1 += 1 * pos
            n2 += 9 * pos
            n -= 10 # to borrow from the place to the left

        elif rem == 1 and n >= 10: # we need to borrow from the left again
            n1 += 2 * pos
            n2 += 9 * pos
            n -= 10

        else: 
	        # less than 10 or remainder = 1, so adding rem - 1 * pos 
	        # just adds 0 or does a valid digit split
            n1 += (rem - 1) * pos
            n2 += 1 * pos
  

        pos *= 10
        n //= 10

    return [n1, n2]
```

Here's a walkthrough of the algorithm using $n = 1012$.

### Setup

$$n = 1012, \quad n_1 = 0, \quad n_2 = 0, \quad \text{pos} = 1$$

At each step we compute:

$$\text{rem} = n \bmod 10$$

and branch on its value.

---

### Iteration 1

$n = 1012$, $\text{rem} = 1012 \bmod 10 = 2$

Since $\text{rem} \geq 2$, this is the simple split case ($\text{rem} - 1$ and $1$):

$$n_1 \mathrel{+}= (\text{rem}-1)\cdot\text{pos} = 1 \cdot 1 = 1 ;\Rightarrow; n_1 = 1$$ $$n_2 \mathrel{+}= 1\cdot\text{pos} = 1 \cdot 1 = 1 ;\Rightarrow; n_2 = 1$$

Update for next round:

$$\text{pos} = 10, \qquad n = \lfloor 1012/10 \rfloor = 101$$

---

### Iteration 2

$n = 101$, $\text{rem} = 101 \bmod 10 = 1$

Since $\text{rem} = 1$ **and** $n \geq 10$, we must borrow from the next place (split into $9$ and $2$ after taking $10$ from the left):

$$n_1 \mathrel{+}= 2\cdot\text{pos} = 2\cdot10 = 20 ;\Rightarrow; n_1 = 21$$ $$n_2 \mathrel{+}= 9\cdot\text{pos} = 9\cdot10 = 90 ;\Rightarrow; n_2 = 91$$

Borrow adjustment:

$$n \mathrel{-}= 10 ;\Rightarrow; n = 91$$

Update for next round:

$$\text{pos} = 100, \qquad n = \lfloor 91/10 \rfloor = 9$$

---

### Iteration 3

$n = 9$, $\text{rem} = 9 \bmod 10 = 9$

$\text{rem} \geq 2$ again, simple split:

$$n_1 \mathrel{+}= (9-1)\cdot\text{pos} = 8\cdot100 = 800 ;\Rightarrow; n_1 = 821$$ $$n_2 \mathrel{+}= 1\cdot\text{pos} = 1\cdot100 = 100 ;\Rightarrow; n_2 = 191$$

Update:

$$\text{pos} = 1000, \qquad n = \lfloor 9/10 \rfloor = 0$$

Loop terminates since $n = 0$.

---

### Result

$$[n_1, n_2] = [821,\ 191]$$

**Verification:**

$$821 + 191 = 1012 = n ;\checkmark$$

- $821$: digits ${8,2,1}$ — no zero ✅
- $191$: digits ${1,9,1}$ — no zero ✅

Note the interesting bit in iteration 2: since $\text{rem}=1$ _and_ $n \geq 10$ (i.e., there are more digits left to process), we can't just split $1$ into $0$ and $1$ (that would create a zero digit). Instead we borrow $10$ from the next higher place, turning that position's contribution into $9 + 2$ — which is exactly what keeps both resulting numbers zero-free.
