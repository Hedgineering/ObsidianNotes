
Given an array of positive integers `arr`, find a pattern of length `m` that is repeated `k` or more times.

A **pattern** is a subarray (consecutive sub-sequence) that consists of one or more values, repeated multiple times **consecutively** without overlapping. A pattern is defined by its length and the number of repetitions.

Return `true` _if there exists a pattern of length_ `m` _that is repeated_ `k` _or more times, otherwise return_ `false`.

**Example 1:**

**Input:** arr = [1,2,4,4,4,4], m = 1, k = 3
**Output:** true
**Explanation:** The pattern **(4)** of length 1 is repeated 4 consecutive times. Notice that pattern can be repeated k or more times but not less.

**Example 2:**

**Input:** arr = [1,2,1,2,1,1,1,3], m = 2, k = 2
**Output:** true
**Explanation:** The pattern **(1,2)** of length 2 is repeated 2 consecutive times. Another valid pattern **(2,1) is** also repeated 2 times.

**Example 3:**

**Input:** arr = [1,2,1,2,1,3], m = 2, k = 3
**Output:** false
**Explanation:** The pattern (1,2) is of length 2 but is repeated only 2 times. There is no pattern of length 2 that is repeated 3 or more times.

**Constraints:**

- `2 <= arr.length <= 100`
- `1 <= arr[i] <= 100`
- `1 <= m <= 100`
- `2 <= k <= 100`

---
## Solution

```py
def containsPattern(self, arr: List[int], m: int, k: int) -> bool:
    i = 0
    cnt = 0

    while i + m < len(arr):  
        if arr[i] != arr[i+m]:
            cnt = 0
        else:
            cnt += 1

        if cnt == (k-1)*m:
            return True

        i += 1

    return False
```

Imagine each collection of $m$ length characters to be one block. For example, if you have $m=1$ and $k=2$, then the array $[1,1]$ would yield `True` because you have one successful comparison of 2 elements. 

Similarly if you had $m=3$ and $k=2$, then $[1,2,1,2,1,2]$ would yield true because each element matches the element at index $i+m$ and (212 == 212). The rule becomes if you are comparing pairwise $i$ and $i+m$ elements, if you have at least $(k-1)*m$ matches then this is `True`.

**Why `(k-1)*m`?** This is the key insight. If a pattern of length `m` repeats `k` times, that spans `k*m` elements total. But we're comparing pairs `(i, i+m)` — think of "links" connecting one block to the next. With `k` blocks, there are `k-1` "links" between consecutive blocks, and each link requires `m` matching positions (one full pattern-length of matches) to confirm the blocks are identical.

So we need `(k-1)*m` consecutive matches of `arr[i] == arr[i+m]` to confirm `k` repeats of a length-`m` pattern.

---
### Example
Here's a walkthrough with for this exact code, using `arr = [1,2,1,2,1,1,1,3]`, `m = 2`, `k = 2`.

#### Setup

$$\text{arr} = [1,2,1,2,1,1,1,3], \quad m = 2, \quad k = 2$$

$$i = 0, \quad \text{cnt} = 0$$

**Target threshold:**

$$\text{cnt} == (k-1)\cdot m = (2-1)\cdot 2 = 2$$

So we're looking for $2$ consecutive matches of $\text{arr}[i] = \text{arr}[i+m]$ before the loop condition $i + m < \text{len(arr)} = 8$ fails.

---

#### Iteration $i = 0$

Check loop condition: $i + m = 0 + 2 = 2 < 8$ ✅

$$\text{arr}[0] = 1, \quad \text{arr}[0+2] = \text{arr}[2] = 1$$

Since $\text{arr}[0] = \text{arr}[2]$:

$$\text{cnt} \mathrel{+}= 1 ;\Rightarrow; \text{cnt} = 1$$

Check: $\text{cnt} = 1 \neq 2$, continue.

$$i \mathrel{+}= 1 ;\Rightarrow; i = 1$$

---

#### Iteration $i = 1$

Check loop condition: $i + m = 1 + 2 = 3 < 8$ ✅

$$\text{arr}[1] = 2, \quad \text{arr}[1+2] = \text{arr}[3] = 2$$

Since $\text{arr}[1] = \text{arr}[3]$:

$$\text{cnt} \mathrel{+}= 1 ;\Rightarrow; \text{cnt} = 2$$

Check: $\text{cnt} = 2 = (k-1)\cdot m$ ✅ **→ return `True`**

---

#### Result

$$\boxed{\text{containsPattern}(\text{arr}, 2, 2) = \text{True}}$$

**Interpretation:** the match at $i=0$ confirms $\text{arr}[0..1] = \text{arr}[2..3]$, i.e. the block $(1,2)$ equals the next block $(1,2)$ — that's the pattern $(1,2)$ repeated $2$ times, consistent with $k=2$.

---

#### Trace table

|$i$|loop cond ($i+m<8$)|$\text{arr}[i]$|$\text{arr}[i+m]$|match?|cnt|cnt == $(k-1)m$?|
|:-:|:-:|:-:|:-:|:-:|:-:|:-:|
|0|$2<8$ ✅|1|1|yes|1|no|
|1|$3<8$ ✅|2|2|yes|2|**yes → True**|

---

#### Bonus: a case that resets `cnt`

To see the reset branch (`arr[i] != arr[i+m]` → `cnt = 0`) in action, consider `arr = [1,2,1,3,1,2,1,2]`, `m = 2`, `k = 2`. Threshold is still $(k-1)\cdot m = 2$.

|$i$|$\text{arr}[i]$|$\text{arr}[i+2]$|match?|cnt|
|:-:|:-:|:-:|:-:|:-:|
|0|1|1|yes|1|
|1|2|3|**no**|**0** (reset)|
|2|1|1|yes|1|
|3|3|2|no|0 (reset)|
|4|1|1|yes|1|
|5|2|2|yes|2 → **True**|

Here you can see `cnt` climbing and getting knocked back to $0$ each time the streak breaks, and it only reaches the threshold once two consecutive matches occur back-to-back — confirming the pattern $(1,2)$ starting at index $4$: $\text{arr}[4..5] = (1,2) = \text{arr}[6..7]$.
