1. Problem Formulation
	- Profits are too low, what should we do?
	- Customers are complaining about long lines.
	
2. Objectives and Planning - define goals and what specific questions to answer
	- How many workers should we hire?
	- How much buffer space should be added to the assembly line?
	
3. Model Building - both an art and a science
	- M/M/k queueing model?
	- Need physics equations?
	
4. Data Collection - what kinds of data do we need, how much to collect (more is usually more expensive) 
	- Continuous? Discrete?
	- What exactly should be collected?
	- Budget considerations.
	
5. Coding - decide language
	- Modeling paradigm - how should you design/architect the simulation framework
		- Event-Scheduling
		- Process-Interaction
	
6. Verification - is the code correct, clean, readable, maintainable, extensible? ; if not, return to step 5
	
7. Validation - is the model OK? correct or good enough? ; if not, return to step 3
	
8. Experimental Design - what experiments need to be run to efficiently answer our questions?
	- Statistical considerations
	- Time / budget
	
9. Run experiments (hit run, wait for simulation and experiments to run their full course)
	
10. Output Analysis - Statistical analysis, estimate relevant performance measures, repeat steps 8 and 9, write reports with full results
