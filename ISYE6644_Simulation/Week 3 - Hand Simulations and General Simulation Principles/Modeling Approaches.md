
# Simulation Modeling Approaches

Discrete-Event Simulation (DES) models a system as a sequence of events that occur at specific points in time. The simulation clock jumps from one event to the next rather than advancing continuously.

Two common DES modeling approaches are:

1. Event Scheduling
2. Process Interaction

---

# Event Scheduling

## Idea

Model the system by explicitly tracking:

- System state variables
- Future Event List (FEL)
- Simulation clock

The simulation repeatedly:

1. Removes the next event from the FEL.
2. Advances the clock to that event time.
3. Executes the event logic.
4. Updates the system state.
5. Schedules any future events generated by that event.

---

## Main Components

### Simulation Clock

$$
t = \text{current simulation time}
$$

The clock jumps directly to the next event time.

---

### System State

Variables describing the current condition of the system.

Examples:

$$
L_Q(t) = \text{number of customers in queue}
$$

$$
B(t) =
\begin{cases}
1 & \text{server busy} \\
0 & \text{server idle}
\end{cases}
$$

---

### Future Event List (FEL)

A time-ordered list of upcoming events.

Example:

$$
\text{FEL}
=
\{
(10,\text{Arrival}),
(12,\text{Departure}),
(15,\text{Arrival})
\}
$$

The next event is always the earliest one.

---

## Generic Event Scheduling Algorithm

```text
Initialize system state
Initialize FEL

while stopping condition not met:

    Remove earliest event from FEL

    Advance clock to event time

    Execute event logic

    Update system state

    Schedule future events
```

---

## Example: Single-Server Queue

Events:

- Arrival
- Departure

### Arrival Event

```text
If server idle:
    Start service immediately
    Schedule departure
Else:
    Join queue

Schedule next arrival
```

---

### Departure Event

```text
If queue non-empty:
    Remove next customer
    Start service
    Schedule departure
Else:
    Set server idle
```

---

## Advantages

- Efficient
- Direct control over events
- Closely follows simulation theory

---

## Disadvantages

- Can become difficult for complex systems
- Must manually manage FEL
- Large amount of bookkeeping

---

# Process Interaction

## Idea

Model the system from the perspective of entities moving through processes.

Instead of explicitly scheduling events, we describe what each entity does.

This is the approach used by simulation software such as Arena.

---

## Concept

Think:

> "What does a customer experience?"

rather than

> "What events occur?"

---

## Customer Process

```text
Arrive

If server busy:
    Wait in queue

Receive service

Depart
```

Each customer follows this process independently.

---

## Example

A customer process might be written conceptually as:

```text
Customer:

    Arrive

    Seize Server

    Delay(Service Time)

    Release Server

    Depart
```

---

## Arena Mapping

| Concept | Arena Module |
|----------|----------|
| Arrival | Create |
| Join Queue | Process (Seize) |
| Service | Delay |
| Leave Server | Release |
| Departure | Dispose |

Typical Arena flow:

```text
Create
   ↓
Process
   ↓
Dispose
```

---

## What Happens Internally?

Even though Arena uses Process Interaction, it still executes an Event Scheduling simulation underneath.

For example:

```text
Customer arrives
    ↓
Arena schedules arrival event

Customer starts service
    ↓
Arena schedules service completion event

Customer departs
    ↓
Arena schedules departure event
```

The FEL still exists internally.

Arena simply hides it from the modeler.

---

# Event Scheduling vs Process Interaction

| Feature | Event Scheduling | Process Interaction |
|-----------|-----------|-----------|
| Focus | Events | Entities |
| User manages FEL | Yes | No |
| Level of detail | Low-level | High-level |
| Easier for large systems | No | Yes |
| Used in Arena | Internally | Yes |
| Typical modeler view | Event logic | Process flow |

---

# Intuition

### Event Scheduling

Think like the simulation engine:

```text
Arrival at t=5
Departure at t=8
Arrival at t=10
...
```

You manage every event explicitly.

---

### Process Interaction

Think like the customer:

```text
Arrive
Wait
Get served
Leave
```

The simulation software automatically converts these actions into events behind the scenes.

---

# Key Exam Takeaway

Event Scheduling:
- Explicitly manage the Future Event List (FEL).
- Update state at every event.

Process Interaction:
- Describe the life cycle of entities.
- Arena uses this approach.
- The simulator automatically generates and manages the FEL internally.
