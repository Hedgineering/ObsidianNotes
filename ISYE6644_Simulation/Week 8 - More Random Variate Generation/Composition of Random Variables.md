
# Composition Method

> [!abstract] Core Idea Suppose a random variable (RV) doesn't come from one "nice" distribution, but is actually a **mixture** of two or more easier RVs "on top of each other." If we can identify the mixture structure, we can generate from the easy pieces instead of the hard whole.

**Motivating example:** A plane's gate departure delay might really be a mix of two causes:

- Air traffic delays
- Maintenance delays

These two "compose" the overall delay distribution. The natural follow-up questions:

- What if there are **many** (not just two) contributing reasons?
- In any case — **how do we actually generate from the mixture?**

This is what the _Composition Method_ answers.

---

## General Setup

The goal is to generate a RV $X$ with a target c.d.f. $F(x)$ that can be written as a **mixture**:

$$ F(x) = \sum_{j=1}^{\infty} p_j F_j(x) $$

> [!note] Don't panic about the $\infty$ The sum can have finitely many terms in practice (e.g. just $j=1,2$) — the upper limit is just written generally.

where:

- $p_j > 0$ for all $j$
- $\sum_j p_j = 1$ (the $p_j$ form a valid probability distribution over the index $j$)
- each $F_j(x)$ is an "easy" c.d.f. that we already know how to sample from

This says: $F$ is a **weighted average (convex combination)** of simpler distributions $F_j$.

---

## Algorithm

1. **Generate a positive integer $J$** such that $P(J = j) = p_j$ for all $j$.
    - (This is itself a discrete-RV generation problem — e.g. inverse transform on the discrete distribution ${p_j}$.)
2. **Return $X$ generated from c.d.f. $F_J(x)$** — i.e., once you know which component $J$ was picked, sample $X$ from that component's (easy) distribution.

```
Step 1: Draw J ~ Discrete(p_1, p_2, ...)
Step 2: Draw X ~ F_J
Step 3: Return X
```

---

## Proof That This Works

We want to show the $X$ generated by the algorithm really has c.d.f. $F(x)$.

By the **law of total probability**, conditioning on which component $J$ was drawn:

$$ P(X \le x) = \sum_{j=1}^{\infty} P(X \le x \mid J = j), P(J = j) $$

Since given $J = j$, $X$ was drawn from $F_j$, we have $P(X \le x \mid J=j) = F_j(x)$, and $P(J=j) = p_j$. So:

$$ P(X \le x) = \sum_{j=1}^{\infty} F_j(x), p_j = F(x). \qquad \blacksquare $$

This confirms the algorithm produces exactly the target mixture distribution.

## Worked Example: Laplace Distribution

The **Laplace distribution** (double exponential) has density and c.d.f.:

$$ 
f(x) = \begin{cases} 
\frac{1}{2} e^{x}, & x < 0 \\
\frac{1}{2} e^{-x}, & x > 0 
\end{cases} 
\qquad F(x) = \begin{cases} 
\frac{1}{2} e^{x}, & x < 0 \\
1 - \frac{1}{2} e^{-x}, & x > 0 
\end{cases} 
$$

> [!tip] Intuition This is just an **exponential distribution reflected off the $y$-axis** — mirrored to also cover negative values.

### Decomposition

Split $X$ into a **"negative exponential"** piece and a **regular exponential** piece:

$$ F_1(x) = \begin{cases} e^{x}, & x < 0 \\ 1, & x > 0 \end{cases} \qquad F_2(x) = \begin{cases} 0, & x < 0 \\ 1 - e^{-x}, & x > 0 \end{cases} $$

- $F_1$: all mass on the negative axis (mirror-image exponential)
- $F_2$: the standard exponential distribution, all mass on the positive axis

### Recombining

$$ F(x) = \frac{1}{2} F_1(x) + \frac{1}{2} F_2(x) $$

So $p_1 = p_2 = \tfrac12$: generate from $F_1$ half the time, and from $F_2$ the other half.

### Generation via Inverse Transform

Solve each piece's inverse transform:

- $F_1(X) = e^{X} = U ;\Rightarrow; X = \ln(U)$
- $F_2(X) = 1 - e^{-X} = U ;\Rightarrow; X = -\ln(U)$

Putting it together:

$$ X \leftarrow \begin{cases} \ln(U), & \text{w.p. } 1/2 \\ -\ln(U), & \text{w.p. } 1/2 \end{cases} \qquad \blacksquare $$

### Recipe Summary

1. Flip a fair coin (generate $J \in {1,2}$ with equal probability).
2. Draw $U \sim \text{Uniform}(0,1)$.
3. If coin says component 1 → return $\ln(U)$; if component 2 → return $-\ln(U)$.

---

## Key Takeaways

- **When to use:** target distribution $F$ can be expressed as a probability-weighted sum of simpler, directly-generatable distributions.
- **Two-step structure:** (1) randomly pick _which_ component distribution to use, weighted by $p_j$; (2) generate from that chosen component.
- **Why it works:** law of total probability — mixing at the generation level exactly reproduces mixing at the c.d.f. level.
- **Generalizes cleanly:** works for any (even infinite) number of mixture components, as long as you can sample the discrete index $J$ and each $F_j$.
- Related/upcoming question: _how do you efficiently generate $J$ itself_ when there are many components? (e.g. alias method, inverse transform on discrete distribution)
